## Introduction

This Work Breakdown Structure (WBS) template provides a hierarchical decomposition of the total scope of work to be carried out by the project team for a construction project. It serves as a framework for scope management, schedule development, cost estimation, and resource allocation.

The WBS is organized with the following structure:
- Level 1: Project Name
- Level 2: Project Phases
- Level 3: Major Deliverables
- Level 4: Work Packages
- Level 5: Activities (optional, often developed in the project schedule)

**Note:** This template represents a comprehensive construction WBS that should be tailored to your specific project type, size, and requirements. Not all sections may apply to every construction project.

---

## WBS Dictionary Guidelines

For each WBS element, consider developing a WBS Dictionary entry that includes:
- WBS ID and Name
- Description of work
- Responsible party/organization
- Required resources
- Estimated duration
- Estimated cost
- Acceptance criteria
- Dependencies
- Quality requirements
- Assumptions and constraints

## WBS Customization Guidelines

### Adapting the WBS for Different Construction Project Types

#### Commercial Buildings
- Emphasize tenant improvement coordination
- Add sections for special commercial systems (e.g., retail, hospitality)
- Expand interior finishes for public spaces

#### Residential Projects
- Simplify MEP systems for residential applications
- Add unit-specific completion milestones
- Include sections for residential amenities

#### Industrial Facilities
- Expand process equipment installation sections
- Add specialized utility systems
- Include equipment testing and validation

#### Infrastructure Projects
- Remove building-specific elements (interiors, etc.)
- Expand earthwork, foundations, and site development
- Add sections for traffic management during construction

### WBS Development Best Practices

1. **Follow the 100% Rule**: The WBS should include 100% of the work defined by the project scope and capture all deliverables â€“ internal, external, and interim.

2. **Use Noun-Based Descriptions**: WBS elements should be described with nouns and adjectives, not verbs. The WBS represents deliverables, not actions.

3. **Apply the 8/80 Rule**: Work packages should require no less than 8 hours and no more than 80 hours of effort. Packages outside this range should be decomposed or consolidated.

4. **Create Mutually Exclusive Elements**: There should be no overlap in scope definition between two elements of a WBS.

5. **Confirm Stakeholder Agreement**: Review the WBS with key stakeholders to ensure all project deliverables are captured.

6. **Leverage Code of Accounts**: Implement a numbering system that clearly indicates the level and relationship of each element.

7. **Use Progressive Elaboration**: Develop the WBS to the level of detail currently known, and refine it as more information becomes available.

8. **Align with Organizational Breakdown Structure (OBS)**: Consider how the work packages align with the organizational units that will perform the work.

## WBS Integration

### Integration with Other Project Management Tools

- **Schedule Development**: Each work package in the WBS becomes an activity or group of activities in the project schedule.
  
- **Cost Estimation**: The WBS provides the framework for developing the cost breakdown structure (CBS).
  
- **Risk Management**: Risks can be identified and managed at the work package level.
  
- **Resource Planning**: Resources can be assigned to specific work packages.
  
- **Quality Management**: Quality requirements and control measures can be defined for each work package.

### WBS and Construction Cost Codes

For effective project control, consider aligning the WBS with standard construction cost codes such as:

- CSI MasterFormat
- Uniformat
- AACE International recommended practices
- Company-specific cost codes

This alignment facilitates cost tracking, earned value management, and integration with accounting systems.
