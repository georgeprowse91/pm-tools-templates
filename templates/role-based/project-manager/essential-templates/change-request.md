# Change Request Template

## Overview

The Change Request Template provides a systematic approach to managing changes to project scope, schedule, budget, or quality. It ensures all changes are properly evaluated, approved, and implemented while maintaining project control.

**Purpose**: Standardize the process for requesting, evaluating, and implementing project changes

**When to Use**: Whenever a proposed change affects project scope, schedule, budget, quality, or resources

**Who Uses**: Anyone can submit; Project Manager manages the process; Change Control Board approves

## Change Management Best Practices

### Proactive Change Management
**Early Identification**:
- Regular stakeholder feedback sessions
- Continuous requirements validation
- Environmental scanning for external changes
- Regular project health checks

**Change Prevention**:
- Thorough upfront planning
- Clear requirements documentation
- Regular stakeholder engagement
- Robust risk management

### Change Evaluation Guidelines
**Evaluation Criteria**:
1. **Strategic Alignment**: Does the change support business objectives?
2. **Value Proposition**: Does the benefit justify the cost/effort?
3. **Risk Assessment**: Are the risks acceptable and manageable?
4. **Resource Availability**: Can the change be resourced appropriately?
5. **Timeline Impact**: Is the schedule impact acceptable?

**Decision Framework**:
- **High Value, Low Risk**: Approve quickly
- **High Value, High Risk**: Detailed analysis required
- **Low Value, Low Risk**: Consider deferring
- **Low Value, High Risk**: Likely reject

### Communication Strategy
**Change Announcement**:
- Clear explanation of what's changing
- Rationale for the change
- Impact on stakeholders
- Implementation timeline
- Support resources available

**Ongoing Communication**:
- Regular progress updates
- Issue escalation paths
- Feedback mechanisms
- Training and support information

**Small Organizations**:
- Simplified approval process
- Informal change board
- Streamlined documentation
- Email-based workflow

**Large Organizations**:
- Formal change control board
- Detailed impact analysis requirements
- Enterprise tool integration
- Formal governance processes

### Project Type Adaptation
**Traditional/Waterfall Projects**:
- Formal change control gates
- Baseline change management
- Detailed documentation requirements
- Sequential approval process

- Lightweight change process
- Sprint-level change decisions
- Product owner authority
- Continuous adaptation

**Hybrid Projects**:
- Scalable change process
