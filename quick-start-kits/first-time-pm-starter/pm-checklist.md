# First-Time PM Checklist

**Essential tasks every new project manager should complete**

## Week 1: Project Foundation

### Project Definition
- [ ] **Understand the problem** - Talk to key stakeholders about why this project exists
- [ ] **Draft project charter** - Use the [simple charter template](project-charter-simple.md)
- [ ] **Get charter approved** - Don't start work without official approval
- [ ] **Set up project folder** - Organize your documents from day one

### Stakeholder Setup
- [ ] **List all stakeholders** - Use the [stakeholder register](stakeholder-register-simple.md)
- [ ] **Meet key stakeholders** - Schedule 30-minute introductions
- [ ] **Understand expectations** - Ask "What does success look like to you?"
- [ ] **Identify project champion** - Find your biggest supporter

### Communication Foundation
- [ ] **Set meeting schedule** - Weekly team meetings minimum
- [ ] **Choose communication tools** - Email, Slack, Teams, etc.
- [ ] **Plan status reporting** - Decide frequency and format
- [ ] **Create distribution lists** - Who gets what information

## Week 2: Planning Essentials

### Team Assembly
- [ ] **Identify team members** - Who will actually do the work?
- [ ] **Define roles clearly** - Use RACI matrix if helpful
- [ ] **Hold kickoff meeting** - Get everyone aligned
- [ ] **Set team communication norms** - How will you work together?

### Risk Management
- [ ] **List obvious risks** - What could go wrong?
- [ ] **Plan risk responses** - How will you handle each risk?
- [ ] **Identify early warning signs** - How will you know risks are happening?
- [ ] **Set up regular risk reviews** - Weekly risk check-ins

### Success Planning
- [ ] **Define "done"** - How will you know when you're finished?
- [ ] **Set quality standards** - What level of quality is expected?
- [ ] **Plan testing/validation** - How will you verify results?
- [ ] **Get approval criteria** - Who decides if deliverables are acceptable?

## Week 3: Execution Preparation

### Project Controls
- [ ] **Set up progress tracking** - How will you monitor work?
- [ ] **Create issue log** - Template for tracking problems
- [ ] **Plan change control** - How will you handle scope changes?
- [ ] **Set up document management** - Where will everything be stored?

### Stakeholder Engagement
- [ ] **Schedule regular check-ins** - Don't let stakeholders disappear
- [ ] **Plan first status report** - Get template ready
- [ ] **Identify escalation path** - Who helps when you're stuck?
- [ ] **Set up feedback mechanisms** - How will stakeholders give input?

## Ongoing (Every Week)

### Project Monitoring
- [ ] **Review progress** - Are you on track?
- [ ] **Update risk register** - Any new risks or changes?
- [ ] **Check team health** - Is anyone struggling or blocked?
- [ ] **Monitor stakeholder satisfaction** - Are people happy with progress?

### Communication
- [ ] **Send status report** - Keep stakeholders informed
- [ ] **Hold team meeting** - Coordinate and align
- [ ] **Update project documentation** - Keep records current
- [ ] **Respond to issues quickly** - Don't let problems fester

### Continuous Improvement
- [ ] **Note lessons learned** - What are you learning?
- [ ] **Adjust processes** - What's working/not working?
- [ ] **Celebrate wins** - Recognize good work
- [ ] **Address problems early** - Don't hope issues will go away

## Red Flags to Watch For üö©

### Stakeholder Issues
- [ ] **Stakeholder stops responding** to communications
- [ ] **Conflicting direction** from different stakeholders
- [ ] **New requirements** being added without approval
- [ ] **Politics or personality conflicts** emerging

### Project Issues
- [ ] **Work taking longer** than expected consistently
- [ ] **Quality problems** in deliverables
- [ ] **Team members leaving** or becoming unavailable
- [ ] **Budget concerns** being raised

### Communication Breakdown
- [ ] **Meetings becoming unproductive** or canceled frequently
- [ ] **Team not collaborating** effectively
- [ ] **Information not flowing** to the right people
- [ ] **Decisions being made** without proper input

## Emergency Checklist üÜò

*When things go wrong, do this immediately:*

1. [ ] **Stop and assess** - Don't panic, understand the situation
2. [ ] **Inform stakeholders** - No surprises, communicate early
3. [ ] **Gather your team** - Get everyone together quickly
4. [ ] **Develop options** - What are possible solutions?
5. [ ] **Make decisions quickly** - Don't overthink, act
6. [ ] **Document everything** - Keep records of what happened
7. [ ] **Learn from it** - Update your risk register

## Success Checklist üéâ

*You'll know you're succeeding when:*

- [ ] **Stakeholders know what's happening** and when it will be done
- [ ] **Team is working well together** and making progress
- [ ] **Problems are caught early** and resolved quickly
- [ ] **You feel confident** about project status and next steps
- [ ] **Others come to you for advice** or want you on their projects

## Quick Reference: Daily Habits

### Every Morning (5 minutes)
- [ ] Review yesterday's progress
- [ ] Check for new issues or emails
- [ ] Plan today's priorities
- [ ] Touch base with team if needed

### Every Friday (30 minutes)
- [ ] Prepare weekly status report
- [ ] Review risks and issues
- [ ] Plan next week's priorities
- [ ] Celebrate week's accomplishments

## Need Help? üÜò

### When You're Stuck
1. **Talk to your project sponsor** - They have authority to help
2. **Consult experienced PMs** - Most are happy to share advice
3. **Review project charter** - Remind yourself of the big picture
4. **Focus on communication** - Most problems are communication problems
