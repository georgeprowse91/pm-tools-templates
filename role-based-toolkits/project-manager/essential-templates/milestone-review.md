# Milestone Review Template

## Overview

The Milestone Review Template provides a structured approach to assess project progress at critical decision points. It ensures thorough evaluation of deliverables, progress against objectives, and readiness to proceed to the next phase.

**Purpose**: Conduct formal assessments of project progress and deliverable quality at key milestones

**When to Use**: At predefined project milestones, phase gates, or critical decision points

**Who Uses**: Project Manager, Steering Committee, Stakeholders, Quality Assurance

## Milestone Review Templates by Type

### Phase Gate Review
**Focus Areas**:
- Phase completion assessment
- Phase deliverable acceptance
- Next phase readiness
- Resource transition planning

**Decision Points**:
- Proceed to next phase
- Repeat current phase activities
- Modify approach for next phase
- Terminate project

### Quality Gate Review
**Focus Areas**:
- Deliverable quality assessment
- Quality standards compliance
- Defect analysis and resolution
- Quality improvement recommendations

**Decision Points**:
- Accept deliverables
- Conditional acceptance with rework
- Reject deliverables
- Modify quality standards

### Go/No-Go Review
**Focus Areas**:
- Business case validation
- Risk assessment
- Resource availability
- Market conditions

**Decision Points**:
- Continue project as planned
- Continue with modifications
- Pause project
- Cancel project

## Customization Guidelines

### Industry Adaptations
**Software Development**:
- Include code quality metrics
- Add security review checkpoints
- Assess technical debt impact
- Review deployment readiness

**Construction/Manufacturing**:
- Include safety assessments
- Add regulatory compliance checks
- Review material delivery status
- Assess environmental impact

**Professional Services**:
- Include client satisfaction metrics
- Review revenue recognition
- Assess resource utilization
- Evaluate scope delivery

### Methodology Alignment
**Traditional/Waterfall**:
- Formal phase gate structure
- Comprehensive documentation requirements
- Sequential approval process
- Detailed milestone criteria

**Agile Projects**:
- Sprint review integration
- Iterative improvement focus
- Flexible success criteria
- Continuous stakeholder feedback

**Hybrid Projects**:
- Risk-based formality
- Adaptive review criteria
- Flexible governance approach
- Scalable assessment process
