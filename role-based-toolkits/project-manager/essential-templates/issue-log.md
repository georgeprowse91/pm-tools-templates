# Issue Log Template

## Overview

The Issue Log is a critical project management tool that captures, tracks, and manages problems and obstacles that arise during project execution. It ensures systematic issue resolution and prevents problems from impacting project success.

**Purpose**: Track and manage all project issues from identification through resolution

**When to Use**: Throughout project execution when problems arise that require attention and resolution

**Who Uses**: Project Manager (owner), Team Members, Stakeholders

## Issue Management Framework

### Issue Categories
- **Technical**: Software bugs, system failures, integration problems
- **Resource**: Team availability, skill gaps, equipment issues
- **Process**: Workflow problems, procedure conflicts, governance issues
- **External**: Vendor delays, regulatory changes, market conditions
- **Quality**: Defects, performance issues, acceptance problems
- **Schedule**: Timeline conflicts, dependency issues, milestone risks
- **Budget**: Cost overruns, funding delays, financial constraints

### Priority Definitions
**Critical (P1)**:
- Project completely blocked
- Major deliverable at risk
- Severe business impact
- Requires immediate attention (within 4 hours)

**High (P2)**:
- Significant project impact
- Important deliverable affected
- Business operations impacted
- Resolution needed within 1-2 days

**Medium (P3)**:
- Moderate project impact
- Workaround available
- Limited business impact
- Resolution needed within 1 week

**Low (P4)**:
- Minimal project impact
- Enhancement or nice-to-have
- No business impact
- Resolution as time permits

### Status Definitions
- **New**: Issue just identified, not yet assigned
- **In Progress**: Someone is actively working on resolution
- **Resolved**: Solution implemented, awaiting verification
- **Closed**: Issue confirmed resolved, no further action needed
- **Escalated**: Moved to higher authority for resolution
- **On Hold**: Resolution paused pending external factor

## Issue Escalation Matrix

### Level 1 - Team Level
**Authority**: Team Lead/Senior Team Member
**Timeframe**: 0-24 hours
**Issue Types**: 
- Technical problems within team expertise
- Minor resource conflicts
- Process clarifications

**Escalation Trigger**: Issue not resolved within 24 hours or exceeds team authority

### Level 2 - Project Manager
**Authority**: Project Manager
**Timeframe**: 1-3 days
**Issue Types**:
- Cross-functional coordination problems
- Resource allocation conflicts
- Minor scope or schedule impacts

**Escalation Trigger**: Issue not resolved within 3 days or affects project objectives

### Level 3 - Project Sponsor
**Authority**: Project Sponsor/Steering Committee
**Timeframe**: 3-5 days
**Issue Types**:
- Significant budget impacts
- Major schedule delays
- Scope change requirements
- Resource conflicts requiring organizational intervention

**Escalation Trigger**: Issue affects project viability or requires executive decision

### Level 4 - Executive Leadership
**Authority**: Senior Management/CEO
**Timeframe**: As needed
**Issue Types**:
- Project cancellation considerations
- Major organizational changes required
- Regulatory or legal implications
- Strategic direction changes

**Escalation Criteria**:
- Project failure imminent
- Significant financial impact (>$[amount])
- Legal or regulatory compliance issues
- Major stakeholder conflicts

## Issue Resolution Processes

### Standard Issue Resolution Process
1. **Issue Identification**
   - Issue discovered and documented
   - Initial impact assessment
   - Priority assignment

2. **Issue Assignment**
   - Assign to appropriate team member
   - Set target resolution date
   - Notify relevant stakeholders

3. **Issue Analysis**
   - Investigate root cause
   - Assess full impact
   - Develop resolution options

4. **Resolution Planning**
   - Select best resolution approach
   - Create action plan with timeline
   - Allocate required resources

5. **Resolution Implementation**
   - Execute action plan
   - Monitor progress
   - Provide regular updates

6. **Resolution Verification**
   - Confirm issue is resolved
   - Validate solution effectiveness
   - Update all stakeholders

7. **Issue Closure**
   - Document final resolution
   - Capture lessons learned
   - Close issue formally

### Critical Issue Process
For Critical (P1) issues:
- **Immediate Response**: Within 1 hour
- **War Room Setup**: If needed for complex issues
- **Continuous Updates**: Every 2-4 hours
- **Executive Notification**: Within 2 hours
- **Post-Incident Review**: Mandatory after resolution

### Escalation Process
1. **Trigger Evaluation**: Determine if escalation criteria met
2. **Escalation Preparation**: Document issue and attempted solutions
3. **Escalation Communication**: Notify next level with full context
4. **Handover**: Transfer ownership to escalated authority
5. **Continued Monitoring**: Original reporter tracks progress
6. **Resolution Communication**: Inform all levels of final resolution

## Issue Tracking Metrics

### Resolution Metrics
**Response Time**: Time from issue creation to assignment
- **Target**: Critical (1 hour), High (4 hours), Medium (1 day), Low (3 days)

**Resolution Time**: Time from assignment to resolution
- **Target**: Critical (4 hours), High (2 days), Medium (1 week), Low (2 weeks)

**First Time Resolution Rate**: Percentage of issues resolved without reopening
- **Target**: >85%

**Escalation Rate**: Percentage of issues requiring escalation
- **Target**: <20%

### Quality Metrics
**Recurrence Rate**: Percentage of issues that recur after resolution
- **Target**: <10%

**Root Cause Identified**: Percentage of issues with documented root cause
- **Target**: >90%

**Prevention Action Taken**: Percentage of issues with prevention measures implemented
- **Target**: >80%

## Issue Log Tools and Integration

### Recommended Tools
**Simple Projects**:
- Excel or Google Sheets with issue tracking template
- Basic project management tools (Trello, Asana)
- Email-based issue tracking

**Medium Projects**:
- Dedicated issue tracking tools (Jira, Azure DevOps)
- Project management platforms with issue modules
- Integrated communication platforms

**Large Projects**:
- Enterprise issue management systems
- Integrated PM/ALM platforms
- Custom workflow automation tools

### Integration with Other PM Tools
**Risk Register Integration**:
- Link issues to risk mitigation failures
- Convert high-impact issues to risks
- Track issue trends as new risk indicators

**Change Control Integration**:
- Link issues requiring scope/schedule changes
- Track change requests generated by issues
- Monitor cumulative impact of issue-driven changes

**Decision Log Integration**:
- Link decisions needed for issue resolution
- Track decision outcomes and implementation
- Monitor decision effectiveness in issue prevention

**Status Reporting Integration**:
- Include key issues in status reports
- Provide issue metrics and trends
- Highlight critical issues requiring attention
