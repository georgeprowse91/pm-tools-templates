### Scoring Guide for Each Factor:

#### 1. Requirements Clarity
1. **Very Low** - Requirements are undefined and will emerge during the project
2. **Low** - High-level vision exists, but details are largely unknown
3. **Moderate** - Core requirements defined, but significant details will emerge
4. **High** - Most requirements well defined with some areas needing refinement
5. **Very High** - Comprehensive, stable requirements with minimal expected changes

#### 2. Tolerance for Change
1. **Very High** - Changes expected and welcomed throughout the project
2. **High** - Changes accommodated with minimal impact on schedule/budget
3. **Moderate** - Some changes expected in defined areas
4. **Low** - Changes discouraged but possible with formal approval
5. **Very Low** - Changes avoided after initial planning phase

#### 3. Delivery Expectations
1. **Continuous Delivery** - Working increments delivered weekly/biweekly
2. **Frequent Increments** - Monthly working deliverables expected
3. **Mixed Approach** - Some incremental deliveries with milestone-based phases
4. **Phase-Based** - Major deliveries at completion of defined phases
5. **Final Delivery** - Complete solution delivered at project end

#### 4. Compliance Needs
1. **Minimal** - Internal standards only, flexible documentation
2. **Low** - Some governance documentation, mostly informal
3. **Moderate** - Balance of documentation and flexibility needed
4. **High** - Formal documentation required for several aspects
5. **Very High** - Strict regulatory/contractual documentation required

#### 5. Stakeholder Involvement
1. **Continuous** - Daily/weekly stakeholder collaboration essential
2. **High** - Regular involvement with direct feedback loops
3. **Moderate** - Mix of ongoing involvement and milestone reviews
4. **Periodic** - Scheduled reviews at key project milestones
5. **Minimal** - Primary engagement at project start and completion

#### 6. Team Experience
1. **Highly Agile** - Team experienced with self-organization and iterative work
2. **Mostly Agile** - Some agile experience, comfortable with adaptation
3. **Mixed** - Experience with both traditional and agile approaches
4. **Mostly Traditional** - Team familiar with defined processes and roles
5. **Highly Traditional** - Team experienced with detailed planning and control

#### 7. Project Complexity
1. **Very High** - Unprecedented, exploratory work with many unknowns
2. **High** - Novel elements with significant technical challenges
3. **Moderate** - Mix of known approaches and some novel elements
4. **Low** - Mostly familiar work with few unique challenges
5. **Very Low** - Routine work using established patterns/processes

#### 8. Project Size/Duration
1. **Very Small** - Small team (1-5 people), 1-3 months
2. **Small** - Team of 5-10 people, 3-6 months
3. **Medium** - Team of 10-20 people, 6-12 months
4. **Large** - Team of 20-50 people, 1-2 years
5. **Very Large** - Multiple teams, 2+ years

#### 9. Budget Constraints
1. **Very Flexible** - Value-driven approach, budget can adapt
2. **Somewhat Flexible** - Some buffer for discovery and adaptation
3. **Moderate** - Defined budget with some flexibility for changes
4. **Tight** - Limited buffer, changes must be carefully managed
5. **Fixed** - Strict budget limitations, minimal flexibility

#### 10. Time Constraints
1. **Very Flexible** - Timeline can adapt based on evolving needs
2. **Somewhat Flexible** - Some deadline flexibility for quality/scope
3. **Moderate** - Key milestones fixed, some flexibility between them
4. **Tight** - Limited flexibility, important deadlines must be met
5. **Fixed** - Strict deadline, timeline cannot shift

## ðŸ”„ Methodology Selection Decision Framework

### 1. Primary Drivers of Methodology Selection

The following factors have the greatest influence on methodology selection:

**High impact factors:**
1. **Requirements certainty** - How well-defined are the deliverables?
2. **Stakeholder engagement model** - How will stakeholders provide input?
3. **Delivery cadence** - What rhythm of delivery provides the most value?
4. **Risk tolerance** - What's the organization's attitude toward uncertainty?
5. **Compliance requirements** - What governance standards must be met?

**Methodology alignment indicators:**

| Factor | Traditional Indicator | Agile Indicator | 
|-------|---------------------|----------------|
| Requirements | Stable from start | Evolve throughout |
| Stakeholders | Prefer oversight | Prefer involvement |
| Delivery | Final handoff | Incremental value |
| Risk attitude | Risk-averse | Risk-tolerant |
| Documentation | Comprehensive | Minimal viable |

### 2. Hybrid Approach Evaluation Criteria

**Consider a hybrid approach when:**

1. **Mixed certainty levels** - Some aspects are well-defined while others will emerge
2. **Varied stakeholder groups** - Some need regular involvement, others milestone reviews
3. **Multiple project components** - Different work streams have different characteristics
4. **Compliance + innovation tension** - Need to satisfy both governance and adaptability
5. **Scaling transition** - Organization moving from traditional to agile practices

**Hybrid composition options:**

1. **Phase-based hybrid** - Traditional planning and initiation, agile execution
2. **Component-based hybrid** - Traditional for stable elements, agile for evolving parts
3. **Layer-based hybrid** - Agile teams with traditional program governance
4. **Ceremony-based hybrid** - Modified ceremonies that blend approaches
5. **Documentation-based hybrid** - Agile delivery with traditional documentation

### 3. Tailoring Your Methodology

No methodology is one-size-fits-all. Consider these tailoring dimensions:

1. **Ceremony frequency** - Adjust meeting cadence based on team needs
2. **Documentation level** - Scale documentation to risk and compliance needs
3. **Planning horizon** - Determine appropriate planning detail and timeframe
4. **Governance rigor** - Align oversight with project risk profile
5. **Team structure** - Design team composition and roles for project needs
