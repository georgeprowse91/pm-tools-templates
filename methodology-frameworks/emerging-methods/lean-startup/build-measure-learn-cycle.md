# Build-Measure-Learn Cycle Template

## Overview
The Build-Measure-Learn feedback loop is the core component of the Lean Startup methodology. This template guides you through each phase of the cycle to maximize validated learning.

- [ ] Build MVP or feature increment

- [ ] Launch product/feature to target segment
- [ ] Monitor user behavior and engagement

- [ ] Compare results to hypothesis

- [ ] Calculate key performance indicators
- [ ] Synthesize quantitative and qualitative data

#### Decision Making
- [ ] Determine if hypothesis was validated
- [ ] Choose next action:

  - **Pivot**: Change fundamental hypothesis

#### Documentation
- [ ] Record lessons learned
- [ ] Update product roadmap
- [ ] Share insights with stakeholders
- [ ] Plan next cycle

## Key Performance Indicators

### Product Metrics
- User acquisition rate
- User engagement/retention
- Feature adoption rate
- User satisfaction score

### Learning Metrics
- Hypothesis validation rate
- Cycle time (Build-Measure-Learn duration)
- Customer interview completion rate
- Experiment success rate

## Common Pitfalls to Avoid

1. **Building too much**: Keep MVPs minimal
2. **Measuring vanity metrics**: Focus on actionable metrics
3. **Learning without action**: Always plan next steps
4. **Skipping customer feedback**: Prioritize qualitative insights
5. **Long cycle times**: Aim for rapid iteration
