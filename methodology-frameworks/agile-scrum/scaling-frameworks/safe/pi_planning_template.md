# SAFe Program Increment (PI) Planning Template

## Overview
This template provides a comprehensive framework for conducting SAFe Program Increment (PI) Planning events, typically held every 8-12 weeks to align multiple Agile Release Trains (ARTs) on a common mission and vision.

## Template Information
- **Framework:** SAFe (Scaled Agile Framework)
- **Event Type:** PI Planning
- **Duration:** 2 days (typical)
- **Participants:** 50-125 people per ART
- **Frequency:** Every 8-12 weeks

---

## Pre-Planning Checklist

### 30 Days Before PI Planning
- [ ] **Vision and Roadmap Preparation**
  - [ ] Product/Solution vision is current and compelling
  - [ ] Program roadmap is updated with latest priorities
  - [ ] Business context is prepared for presentation
  - [ ] Top 10 features are prioritized and refined

- [ ] **Logistics and Setup**
  - [ ] Venue booked (or virtual platform prepared)
  - [ ] All teams confirmed attendance
  - [ ] Facilitation materials prepared
  - [ ] Technology setup tested (projectors, wifi, collaboration tools)

- [ ] **Content Preparation**
  - [ ] Program backlog is prioritized and refined
  - [ ] Architectural runway items identified
  - [ ] Dependencies mapped between teams
  - [ ] Capacity calculations completed

### 1 Week Before PI Planning
- [ ] **Final Preparations**
  - [ ] PI Planning agenda finalized and distributed
  - [ ] Team breakout spaces assigned
  - [ ] Parking lot boards prepared
  - [ ] Confidence vote materials ready
  - [ ] Risk/impediment tracking boards set up

## Team Breakout Session Template

### Team Breakout Structure
Each team should follow this structure during breakout sessions:

#### Session 1: Story & Dependency Identification
- **Duration:** 1 hour 45 minutes
- **Objective:** Identify and size stories, map dependencies

**Activities:**
1. **Story Identification (45 minutes)**
   - Review top 10 features
   - Identify user stories for the PI
   - Estimate story points
   - Consider capacity constraints

2. **Dependency Mapping (45 minutes)**
   - Identify dependencies on other teams
   - Map external dependencies
   - Note assumptions and risks
   - Update dependency board

3. **Initial Planning (15 minutes)**
   - Draft iteration plans
   - Identify major milestones
   - Note questions for parking lot

#### Session 2: Plan Refinement
- **Duration:** 1 hour 30 minutes
- **Objective:** Refine plans based on feedback and dependencies

**Activities:**
1. **Dependency Resolution (30 minutes)**
   - Review feedback from dependency board
   - Adjust plans based on other team commitments
   - Negotiate critical dependencies

2. **Capacity Planning (45 minutes)**
   - Validate team capacity
   - Account for holidays, training, etc.
   - Adjust story commitments
   - Plan innovation and planning time

3. **Risk Assessment (15 minutes)**
   - Identify technical risks
   - Note business risks
   - Assess dependency risks
   - Update risk board

#### Session 3: Final Planning
- **Duration:** 1 hour 45 minutes
- **Objective:** Finalize PI commitments and prepare presentation

**Activities:**
1. **Final Adjustments (45 minutes)**
   - Incorporate management feedback
   - Resolve remaining dependencies
   - Finalize story commitments
   - Complete iteration planning

2. **Presentation Preparation (30 minutes)**
   - Prepare team PI objective slides
   - Practice presentation
   - Identify spokesperson
   - Prepare for Q&A

3. **Confidence Assessment (30 minutes)**
   - Assess team confidence in plan
   - Identify remaining risks
   - Prepare confidence vote rationale
   - Update commitment statements

## Program Board Template

### Board Structure
Create a physical or digital board with:

**Rows:** Teams (one row per team)
**Columns:** Iterations (typically 5 iterations per PI)

### Content Guidelines

#### Features and Stories
- **Blue stickies:** Features
- **Yellow stickies:** Stories
- **Red stickies:** Milestones
- **Green stickies:** Dependencies

#### Dependencies
- **Red yarn/lines:** Dependencies between teams
- **Dotted lines:** External dependencies
- **Thick lines:** Critical path dependencies

## Confidence Vote Template

### Confidence Vote Process

**Instructions:** Each participant votes on their confidence in achieving the PI objectives using a fist-of-five vote.

#### Confidence Scale
- **5 - High Confidence:** I'm confident we can achieve our objectives
- **4 - Confident:** I'm confident with minor concerns
- **3 - Medium Confidence:** I have some concerns but think we can do it
- **2 - Low Confidence:** I have serious concerns
- **1 - Very Low Confidence:** I don't think we can achieve the objectives
- **0 - No Confidence:** I don't believe this plan will work

#### Vote Tracking Template

| Team | Vote 1 | Concerns | Adjustments Made | Vote 2 | Final |
|------|--------|----------|------------------|--------|--------|
| Team Alpha | 3 | [Concerns listed] | [Adjustments] | 4 | ✓ |
| Team Beta | 2 | [Concerns listed] | [Adjustments] | 4 | ✓ |
| Team Gamma | 4 | [Minor concerns] | [Minor adjustments] | 4 | ✓ |

### Confidence Vote Results
- **Target:** All teams at 3 or higher
- **Actual Results:** [Record final votes]
- **Action Items:** [Any remaining items to address]

## Post-PI Planning Activities

### Immediate Actions (Within 24 Hours)
- [ ] Distribute final PI objectives to all stakeholders
- [ ] Update program board with final commitments
- [ ] Share RAID log with all teams
- [ ] Schedule dependency coordination meetings
- [ ] Send thank you note to all participants

### Within 1 Week
- [ ] Load PI objectives into tracking tools (Jira, Azure DevOps, etc.)
- [ ] Set up iteration planning meetings for all teams
- [ ] Establish ART sync meeting cadence
- [ ] Create PI dashboard for tracking progress
- [ ] Schedule mid-PI checkpoint meeting

### Ongoing Activities
- [ ] Weekly ART sync meetings
- [ ] Bi-weekly dependency coordination
- [ ] Monthly stakeholder updates
- [ ] Quarterly PI planning preparation
- [ ] Continuous improvement based on retrospectives

## Customization Guidelines

### Scaling Considerations
- **Single ART (50-125 people):** Use full template as provided
- **Multiple ARTs:** Add solution-level coordination activities
- **Remote/Hybrid:** Emphasize digital collaboration tools
- **New to SAFe:** Include additional coaching and explanation time

### Industry Adaptations
- **Software Development:** Emphasize technical dependencies and architecture
- **Hardware/Manufacturing:** Include supplier dependencies and longer lead times
- **Financial Services:** Add compliance and regulatory considerations
- **Healthcare:** Include patient safety and regulatory requirements
